# SkyTunes 乐谱管理与安全监控系统 — 项目技术分享

> 一个集乐谱管理、在线监控、安全防御与外部数据代理于一体的 PHP 后端系统

---

## 项目背景

SkyTunes 是一个为特定音乐类 App 设计的服务端项目，主要提供乐谱数据的存储、查询与加密传输，同时附带在线用户统计、异常请求监控、IP 自动拉黑、邮件报警以及第三方游戏数据查询（光遇身高）等扩展功能。整个系统采用 PHP 原生开发，前后端分离，管理界面使用 Vue 3 构建。

本文将从**业务模块**、**数据库设计**、**安全机制**、**技术亮点**和**部署建议**五个维度，详细介绍该项目的架构与实现。

---

## 一、核心业务模块

### 1. 乐谱数据管理

#### 1.1 乐谱上传（批量）
- **入口**：`updata.php`（API） + `updata.html`（管理界面）
- **流程**：
  - 管理员在 Web 端选择多个 `.txt` 文件，每个文件包含一个或多个乐谱的 JSON 对象。
  - 提交时需要附带 `admin_key`（预生成的上传密钥）。
  - 后端校验密钥（从 `key_list.json` 读取），检查文件格式，并确保乐谱的 `name` + `author` 组合不重复。
  - 解析 JSON 后批量插入 `Music_Sheet_Data` 表。
- **交互体验**：
  - 支持多文件选择，实时显示已选文件列表。
  - 上传过程展示进度条（总进度 + 当前成功数）。
  - 最终反馈成功/失败明细。

#### 1.2 乐谱查询（明文接口）
- **入口**：`list_music.php`
- **请求参数**：`name`（曲名）、`signature`（签名）、`timestamp`、`nonce`
- **安全校验**（详见下文安全章节）：
  - 请求来源必须为 App 端（通过 `User-Agent` 识别）。
  - 时间戳窗口限制（30 秒）。
  - 签名有效性校验。
- **响应**：返回该曲目的完整数据（作者、BPM、音符数组等）。

#### 1.3 乐谱查询（加密接口）
- **入口**：`YY.php`
- 与明文接口逻辑相同，但响应数据经过**双层 AES-256-CBC 加密 + HMAC 校验**，同时返回 `iv` 和 `salt`，供客户端解密使用。此方式可有效防止中间人窃取乐谱内容。

#### 1.4 乐谱名称列表
- **入口**：`music.php` / `music_list.php`
- 返回数据库所有乐谱名称，供前端展示乐谱库，支持搜索与分页。

### 2. 在线用户统计

系统提供了一套简洁的在线人数统计机制：

- **`Online_users.php`**：每次 App 请求时调用，根据客户端 IP 更新最后访问时间。若用户为新 IP 或已离线，则重新记录上线时间，并插入一条日活记录。
- **`update_user_status.php`**：建议通过 Cron 定时执行（如每 5 分钟），将超过 300 秒未活动的用户标记为离线并删除记录，保持在线表精简。
- **`读取在线人数.php`**：独立接口，仅查询当前在线人数，供前端轮询显示。

> **注**：当前以 IP 作为用户标识，后续可替换为设备 ID 或 Token 以提升准确性。

### 3. 安全监控与自动防御

#### 3.1 IP 黑名单
- 表 `blacklisted_ips` 存储被拉黑的 IP 及原因。
- 所有业务接口在处理前均会调用 `check_blacklist.php` 或内联检查，若命中则直接拒绝。

#### 3.2 请求日志与自动拉黑
- 表 `request_logs` 记录每个 IP 的非法请求次数（`illegal_requests`）和未经授权请求次数（`unauthorized_requests`）。
- 当某一 IP 的累计次数超过 10 次时，系统会自动将其加入黑名单，并清空该 IP 的日志记录。
- 此机制有效抵御暴力破解、扫描或异常客户端。

#### 3.3 邮件报警
- **入口**：`mailbox.php`
- 当检测到严重异常（如非法请求、签名失败等），系统可通过邮件即时通知管理员。
- 邮件内容包含报警事件、错误码、违规 IP 和时间，便于快速响应。
- 同时，违规记录会存入 `illegal_vpn_usage` 表，用于后续审计。

---

### 4. 外部数据代理（光遇身高查询）

为了丰富 App 功能，系统集成了第三方游戏数据查询服务：

- **`Skycnheight.php`**：
  - 接收 `id`（玩家 ID）、`card_key`（卡密）、时间戳和签名。
  - 校验卡密有效性（使用次数限制），通过后调用外部 API 获取玩家身高数据。
  - 对返回数据做规范化处理，并附带剩余使用次数，统一返回给客户端。
- **`skyquery.php`**：通用代理，根据 `linkId` 选择不同的外部 API（支持图片或 JSON 响应），直接透传结果，无需额外鉴权（适用于公开数据）。

---

### 5. 管理员后台（密钥生成器）

- **入口**：`key_gen.php`
- 功能：管理员登录（固定密码）后，可生成、删除、清空或导出上传密钥（18 位字符串）。
- 密钥存储在 `key_list.json` 中，供上传接口校验使用。
- 此设计将上传权限与业务逻辑分离，便于权限控制。

## 二、数据库设计概览

| 表名 | 用途 |
|------|------|
| `Music_Sheet_Data` | 存储乐谱完整信息（曲名、作者、BPM、音符 JSON 等） |
| `online_users` | 记录在线用户 IP 及最后活动时间 |
| `daily_online_sessions` | 每日登录日志，用于统计日活 |
| `blacklisted_ips` | 被拉黑的 IP 列表 |
| `offline_ips` | 手动下线的 IP（预留字段） |
| `request_logs` | 各 IP 的非法/未授权请求计数 |
| `illegal_vpn_usage` | 违规事件记录（用于邮件报警来源） |
| `card_keys` | 身高查询卡密及使用次数 |

## 三、安全机制详解

| 安全策略 | 实现方式 |
|----------|----------|
| **请求签名** | `md5(参数 + SECRET_KEY + timestamp + nonce)`，防止数据篡改 |
| **时间戳窗口** | 限制请求与服务器时间差在 10~30 秒内，防御重放攻击 |
| **请求来源识别** | 检查 `User-Agent` 是否包含特定标识（如 `MyAppUserAgent`），拒绝第三方调用 |
| **IP 黑名单 + 自动拉黑** | 累计错误次数超限自动加入黑名单 |
| **上传密钥校验** | 预生成密钥列表，上传时比对 |
| **卡密使用限制** | 外部查询接口采用卡密机制，防止滥用 |
| **数据加密传输** | 敏感数据（乐谱内容）使用 AES-256 双重加密 + HMAC |
| **邮件报警** | 关键异常实时通知管理员 |

## 四、技术亮点

1. **前后端分离**  
   管理界面使用 Vue 3 构建，调用后端 REST 风格 API，界面美观、交互流畅。

2. **毛玻璃 UI 设计**  
   管理后台采用 Glassmorphism 设计语言，视觉体验现代。

3. **批量上传与进度反馈**  
   支持多文件异步上传，实时显示进度条，提升管理员操作效率。

4. **双层加密机制**  
   乐谱数据在传输前先进行自定义偏移+异或加密，再进行双重 AES-256-CBC 加密，并附加 HMAC，安全性较高。

5. **自动化防御闭环**  
   从请求日志、计数、拉黑到报警，形成完整的自动化安全闭环。

6. **灵活的数据代理**  
   `skyquery.php` 可根据参数动态选择不同外部 API，扩展性强。

## 五、部署注意事项

- **环境要求**：PHP 7.4+，需开启 `mysqli`、`curl`、`openssl`、`json` 等扩展。
- **第三方依赖**：PHPMailer 需通过 Composer 安装。
- **文件权限**：确保 `key_list.json`、`cishu.txt` 以及上传目录可读写。
- **定时任务**：建议设置 Cron 每 5 分钟执行 `update_user_status.php`。
- **敏感信息保护**：代码中的数据库密码、SMTP 密码、SECRET_KEY 等应迁移至环境变量或配置文件，避免硬编码。
- **日志管理**：`logfile.log` 需定期归档，防止占满磁盘。

## 六、总结与展望

SkyTunes 项目是一个功能较为完整的服务端实践，涵盖了业务管理、安全防护、监控报警和外部集成等多个维度。它展示了如何用简洁的 PHP 代码构建一个具备一定安全强度的 App 后端系统。

未来可优化方向：
- 使用 JWT 或 OAuth2 替代自定义签名，提升标准化。
- 引入 Redis 缓存在线用户数据，减少数据库压力。
- 增加 API 版本管理，便于迭代。
- 将敏感配置抽离至 `.env` 文件。

该项目代码结构清晰，适合作为中小型项目参考，也可用于教学演示安全编程实践。
